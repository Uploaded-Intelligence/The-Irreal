## §3 VISUAL LANGUAGE

### Shapes & Geometry
- **Sacred geometry** — circles within circles, vesica piscis, golden spirals
- **Non-Euclidean hints** — geometry that's almost-but-not-quite-right
- **Organic curves meeting geometric precision** — Orokin style
- **Fractal depth** — patterns that repeat at different scales
- **Grown-not-built** — structures that look cultivated, not constructed

### Textures
- **NOT clean/smooth** — weathered, ancient, lived-in
- **Subtle noise/grain** — like old film, like stone
- **Bioluminescent veins** — organic light sources embedded in dark
- **Particle fields** — dust, spores, motes of light
- **Atmospheric haze** — volumetric fog, depth layers

### Typography
- **Crystallizing** — text that forms from particles, not fades in
- **Ornate serifs** — Cormorant Garamond, EB Garamond
- **Slight glow** — text emits subtle light
- **Weight indicates importance** — evergreen content heavier

### Iconography
- **Mythopoetic symbols** — eyes, trees, roots, gates, spirals
- **NOT generic sci-fi** — no hexagons, no circuit patterns
- **Alchemical/occult** — symbols that feel like they MEAN something

## §4 SOUND DESIGN

### Threshold Ambient
- **Sub-bass drone** (40-80Hz) — felt more than heard
- **Breath rhythm** — world breathing at 60 BPM (resting heart rate)
- **Crystalline chimes** — sparse, low-pitched, not bright/cheerful
- **Distant resonance** — sounds from impossibly far away
- **Organic undertones** — heartbeat, membrane, flowing

### Stage Progression
| Stage | Sound Character |
|-------|-----------------|
| Detection | Silence → first sub-bass hint |
| Void | Deep drone, slow pulse |
| Attunement | Drone rises (40→80Hz), responds to input |
| Crystallization | Ice-forming sounds, harmonic resolution |
| Portals | Bleed-through from destinations, spatial audio |
| Crossing | Doppler shift, sound enveloping |

### Generative Principles
- **Never loops** — infinite variation via Tone.js
- **Responds to interaction** — mouse movement affects harmonics
- **Crossfades between states** — never jarring transitions

## §5 THRESHOLD STAGES (Redesigned)

### Stage 0: Detection (Pre-load)
**Duration**: Before visible content

**What Happens**:
- Detect system time, motion preferences, audio capability
- Prepare personalized entrance (returning visitor vs. first time)
- Load ambient soundscape
- Black screen with subtle pulse (barely perceptible)

**Visuals**: Pure darkness, perhaps one distant point of light

**Sound**: Silence transitioning to first sub-bass whisper

---

### Stage 1: The Void (0-3s)
**Duration**: 3 seconds

**What Happens**:
- Darkness becomes alive
- Particle field emerges — NOT purple dots, but:
  - Bioluminescent spores drifting
  - Distant lights like stars but organic
  - Sacred geometry hints forming and dissolving
- Particles pulse at 60 BPM (breathing rhythm)
- Subtle attraction to cursor (the void notices you)

**Visuals**:
- Deep void gradient (not flat black)
- Particles: mixture of scales (tiny dust + distant lights)
- Faint geometry — circles, spirals appearing in negative space
- Volumetric fog/haze layers
- Color: Abyssal Black → hints of Deep Amethyst + Ethereal Teal

**Sound**: Sub-bass drone (40Hz), breathing undertone

**Technical**:
- Particle system with multiple scales
- Perlin noise for organic movement
- Subtle bloom on brighter particles
- Fog shader for depth

---

### Stage 2: Attunement (3-5s)
**Duration**: 2 seconds

**What Happens**:
- Particles begin synchronizing with YOUR rhythm
- Mouse movement → particle velocity
- Your presence affects the space
- The world is learning you

**Visuals**:
- Particles respond more strongly to input
- Sacred geometry becomes more defined
- Hint of structure emerging from void
- Colors warming slightly — gold hints appearing

**Sound**: Drone rises (40→60→80Hz), harmonics respond to movement

**Technical**:
- Velocity tracking on mouse
- Particles accelerate/decelerate based on input
- Geometry shader for sacred geometry hints

---

### Stage 3: Crystallization (5-8s)
**Duration**: 3 seconds

**What Happens**:
- "THE IRREAL" emerges from void
- NOT fade-in — CRYSTALLIZES
- Particles converge to form letterforms
- Each letter built from light coalescing
- Typography pulses with your detected rhythm

**Visuals**:
- Particles streaming toward text position
- Letters forming as particles lock into place
- Subtle glow aura around text
- Sacred geometry frame around title (optional)
- Color: Void White text with Pale Gold glow

**Typography**:
- Cormorant Garamond, light weight
- Large scale (fills significant portion of view)
- Letter-spacing wide, breathing room
- Slight texture/grain in the letterforms

**Sound**: Crystallization sounds — ice forming, harmonic resolution, chimes

**Technical**:
- GSAP for text animation
- Particle system targeting letter positions
- Bloom/glow post-processing

---

### Stage 4: The Portals (8-12s)
**Duration**: 4 seconds

**What Happens**:
- Two portals materialize below title
- NOT geometric rings — ORGANIC FORMS
- Living gateways, breathing, pulsing
- Each shows hint of destination
- Sound bleed-through from each world

**Portal Design**:
Instead of torus geometry:
- **Organic apertures** — like eyes opening, or flowers blooming
- **Orokin-style frames** — ornate, gold-accented, curved
- **Bioluminescent edges** — pulsing with ethereal light
- **Interior shows preview** — glimpse of destination (particles/color)

**The Grove Portal** (left):
- Warmer tones (Arterial Rose + Ancient Bronze)
- Organic, growing, root-like frame
- Interior hints: gentle upward drift, green-gold

**The Atlas Portal** (right):
- Cooler tones (Ethereal Teal + Moonlit Silver)
- Geometric but organic, constellation-like frame
- Interior hints: network connections, node lights

**Hover Behavior**:
- Portal breathes faster
- Sound from that world increases
- Glow intensifies
- Frame ornaments animate

**Sound**: Ambient bleed from each destination, spatial positioning

**Technical**:
- Custom portal geometry (NOT torus)
- Shader for interior preview effect
- Spatial audio positioning
- Hover state animations

---

### Stage 5: The Crossing (on click)
**Duration**: 2 seconds

**What Happens**:
- Clicked portal EXPANDS to consume screen
- You move THROUGH (z-space dolly)
- Not page transition — continuous space
- Sound transitions (crossfade)
- Particles respond (rushing past)

**Visuals**:
- Portal grows from center
- View moves INTO portal
- Particles stream past (speed lines but organic)
- Destination world emerges
- Color palette shifts to destination

**Sound**: Doppler shift, whoosh, destination ambient rising

**Technical**:
- Camera dolly animation
- Particle velocity increase (rushing past)
- Color grade transition
- Seamless to next page/component

## §6 SHADER & WEBGL TECHNIQUES

### Void Atmosphere
```
- Volumetric fog (ray marching)
- Multi-layer gradients with noise
- Subtle chromatic aberration at edges
- Film grain overlay
```

### Particles
```
- GPU instancing for performance
- Multiple particle systems at different scales:
  - Micro: dust/spores (thousands, tiny)
  - Meso: motes of light (hundreds, small)
  - Macro: distant stars (dozens, larger)
- Perlin noise for organic movement
- Attraction forces toward cursor
- Breathing scale animation (synchronized)
```

### Sacred Geometry
```
- Line-based geometry (not filled shapes)
- Opacity animation (fade in/out)
- Rotation over time
- Multiple layers at different depths
```

### Portals
```
- Custom geometry (organic shapes, not primitives)
- Fresnel glow at edges
- Interior: shader showing destination preview
- Animated ornaments on frame
```

### Post-Processing
```
- Bloom (subtle, not overwhelming)
- Vignette (draws focus to center)
- Color grading (per-stage)
- Subtle noise/grain
```

## §11 AI ASSET PRODUCTION PIPELINE

### Overview

Modern creative technologists use AI-assisted pipelines to generate game-quality assets for immersive web experiences. This replaces traditional weeks-long asset creation with hours.

**Pipeline**: `Imagen → Tripo AI → Blender (optional) → Three.js`

### Phase 1: Concept Generation (Imagen/Gemini)

**Goal**: Generate 2D reference art for portals, atmospheres, sacred geometry

**Tools**: Google Imagen 3 / Gemini Image Generation

**Outputs**:
- Portal concept designs (multiple variations)
- Void atmosphere references
- Sacred geometry patterns
- Particle/texture sprites
- Color palette validation

**Prompt Strategy** (for Imagen):
```
Structure: [Subject] + [Style references] + [Mood] + [Technical specs]

Example prompts:

PORTAL CONCEPTS:
"Organic portal gateway, Orokin aesthetic from Warframe, gold and deep purple
bioluminescent edges, Art Nouveau ornamental frame, Moebius dreamscape influence,
ethereal void visible through aperture, front view, concept art, 16:9"

VOID ATMOSPHERE:
"Cosmic void with floating sacred geometry, vesica piscis and golden spirals,
bioluminescent particles like spores, Numenera aesthetic, vast scale,
melancholy wonder not horror, John Harris influence, 16:9"

TEXTURE SPRITES:
"Soft glow particle sprite, bioluminescent spore, ethereal teal and amethyst,
transparent background, centered, sprite sheet ready, 1:1"

SACRED GEOMETRY:
"Intricate sacred geometry mandala, gold lines on void black, vesica piscis
nested circles, Art Deco precision meets organic flow, Monstress detail level,
transparent background, 1:1"
```

### Phase 2: 2D → 3D Conversion (Tripo AI)

**Goal**: Convert concept art to 3D models

**Tool**: [Tripo AI](https://www.tripo3d.ai/) (Image-to-3D in ~10 seconds)

**Workflow**:
1. Upload best Imagen portal concept
2. Generate 3D model (automatic)
3. Use "Build and Refine" for retopology
4. Export as GLB (web-optimized)

**Settings**:
- Enable PBR materials
- Request clean topology
- Set appropriate poly count for web

**Outputs**:
- `portal-grove.glb` — The Grove portal 3D model
- `portal-atlas.glb` — The Atlas portal 3D model
- `sacred-geometry.glb` — Floating geometry elements

### Phase 3: Texture Generation (Imagen)

**Goal**: Generate textures for particles, noise, glows

**Types needed**:
```
1. PARTICLE SPRITES
   - Soft glow sprites (various sizes)
   - Spore/dust particles
   - Star/distant light particles

2. NOISE TEXTURES
   - Void noise (for depth)
   - Atmospheric haze
   - Film grain overlay

3. PORTAL INTERIORS
   - Grove preview texture (warm, organic)
   - Atlas preview texture (cool, network)
```

**Export specs**:
- PNG with transparency where needed
- Powers of 2 dimensions (512x512, 1024x1024)
- Seamless tiling for noise textures

### Phase 4: Animation References (Veo)

**Goal**: Generate video references for transitions and effects

**Tool**: Google Veo 3.1

**Uses**:
- Portal opening/closing animation reference
- Crossing transition effect reference
- Particle flow patterns
- Can be used as video textures in Three.js

**Prompt examples**:
```
"Portal aperture opening like an eye, organic iris, gold and purple
bioluminescent edges, void visible inside, smooth 2 second animation"

"Particles rushing past camera, entering portal, speed lines but organic,
ethereal teal and amethyst, first person perspective"
```

### Phase 5: Refinement (Blender - Optional)

**When needed**:
- Tripo output needs topology cleanup
- Combining multiple 3D elements
- Baking complex materials to simple textures
- Creating animation-ready rigs

**Blender workflow**:
1. Import GLB from Tripo
2. Decimate/retopologize if needed
3. Repack UVs
4. Bake textures (normals, AO)
5. Export clean GLB for Three.js

### Phase 6: Web Integration (Three.js)

**Load assets**:
```typescript
import { useGLTF } from '@react-three/drei';

// Load portal model
const portalModel = useGLTF('/assets/portal-grove.glb');

// Load particle texture
const particleTexture = useTexture('/assets/particle-sprite.png');

// Load noise for atmosphere
const noiseTexture = useTexture('/assets/void-noise.png');
```

**Apply to existing components**:
- Replace `<torusGeometry>` with loaded GLB models
- Replace `<pointsMaterial>` with textured sprites
- Add noise textures to fog shaders
